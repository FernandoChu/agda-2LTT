Idea

The {{#concept "equality" Agda=Id}} relation is defined in type theory by the {{#concept "identity type" Agda=Id}}. The identity type on a type A is a binary family of types

  Id : A ‚Üí A ‚Üí ùí∞

equipped with a {{#concept "reflexivity element" Disambiguation="identity type" Agda=refl}}

  refl : (x : A) ‚Üí Id x x.

In other words, the identity type is a reflexive type valued relation on A. Furthermore, the identity type on A satisfies the universal property that it maps uniquely into any other reflexive relation.

In type theory, we introduce the identity type as an inductive family of types, where the induction principle can be understood as expressing that the identity type is the least reflexive relation.

Notation of the identity type

We include two notations for the identity type. First, we introduce the identity type using Martin-L√∂f's original notation Id. Then we introduce as a secondary option the infix notation _Ôºù_.

Note: The equals sign in the infix notation is not the standard equals sign on your keyboard, but it is the full width equals sign. Note that the full width equals sign is slightly wider, and it is highlighted like all the other defined constructions in Agda. In order to type the full width equals sign in Agda's Emacs Mode, you need to add it to your agda input method as follows:

  • Type M-x customize-variable and press enter.
  • Type agda-input-user-translations and press enter.
  • Click the INS button
  • Type the regular equals sign = in the Key sequence field.
  • Click the INS button
  • Type the full width equals sign Ôºù in the translations field.
  • Click the Apply and save button.

After completing these steps, you can type \= in order to obtain the full width equals sign Ôºù.

Definition

Identity types

We introduce identity types as a data type. This is Agda's mechanism of introducing types equipped with induction principles. The only constructor of the identity type Id x : A ‚Üí ùí∞ is the reflexivity identification

  refl : Id x x.

module _
  {l : Level} {A : UU l}
  where

  data Id (x : A) : A ‚Üí UU l where
    instance refl : Id x x

  infix 6 _Ôºù_
  _Ôºù_ : A ‚Üí A ‚Üí UU l
  (a Ôºù b) = Id a b

{-# BUILTIN EQUALITY Id #-}

We marked refl as an instance to enable Agda to automatically insert refl in definitions that make use of Agda's instance search mechanism.

Furthermore, we marked the identity type as BUILTIN in order to support faster type checking.

The induction principle of identity types

The induction principle of identity types states that given a base point x : A and a family of types over the identity types based at x, B : (y : A) (p : x Ôºù y) ‚Üí UU l2, then to construct a dependent function f : (y : A) (p : x Ôºù y) ‚Üí B y p it suffices to define it at f x refl.

Note that Agda's pattern matching machinery allows us to define many operations on the identity type directly. However, sometimes it is useful to explicitly have the induction principle of the identity type.

ind-Id :
  {l1 l2 : Level} {A : UU l1}
  (x : A) (B : (y : A) (p : x Ôºù y) ‚Üí UU l2) ‚Üí
  (B x refl) ‚Üí (y : A) (p : x Ôºù y) ‚Üí B y p
ind-Id x B b y refl = b

Operations on the identity type

The identity types form a weak groupoidal structure on types. Thus they come equipped with {{#concept "concatenation" Disambiguation="identifications" Agda=concat}} (x Ôºù y) ‚Üí (y Ôºù z) ‚Üí (x Ôºù z) and an {{#concept "inverse" Disambiguation="identification" Agda=inv}} operation (x Ôºù y) ‚Üí (y Ôºù x).

There are many more operations on identity types. Some of them are defined in path algebra and whiskering of identifications. For a complete reference to all the files about general identity types, see the table given above.

Concatenation of identifications

The {{#concept "concatenation operation on identifications" Agda=‚àô Agda=‚àô' Agda=concat}} is a family of binary operations

  _‚àô_ : x Ôºù y ‚Üí y Ôºù z ‚Üí x Ôºù z

indexed by x y z : A. However, there are essentially three different ways we can define concatenation of identifications, all with different computational behaviours.

  1. We can define concatenation by induction on the equality x Ôºù y. This gives us the computation rule refl ‚àô q ‚âê q.
  2. We can define concatenation by induction on the equality y Ôºù z. This gives us the computation rule p ‚àô refl ‚âê p.
  3. We can define concatenation by induction on both x Ôºù y and y Ôºù z. This only gives us the computation rule refl ‚àô refl ‚âê refl.

While the third option may be preferred by some for its symmetry, for practical reasons, we prefer one of the first two, and by convention we use the first alternative.

The second option is considered in foundation.strictly-right-unital-concatenation-identifications, and in foundation.yoneda-identity-types we construct an identity type which satisfies both computation rules among others.

module _
  {l : Level} {A : UU l}
  where

  infixl 15 _‚àô_
  _‚àô_ : {x y z : A} ‚Üí x Ôºù y ‚Üí y Ôºù z ‚Üí x Ôºù z
  refl ‚àô q = q

  concat : {x y : A} ‚Üí x Ôºù y ‚Üí (z : A) ‚Üí y Ôºù z ‚Üí x Ôºù z
  concat p z q = p ‚àô q

  concat' : (x : A) {y z : A} ‚Üí y Ôºù z ‚Üí x Ôºù y ‚Üí x Ôºù z
  concat' x q p = p ‚àô q

Concatenating with inverse identifications

module _
  {l : Level} {A : UU l}
  where

  inv-concat : {x y : A} (p : x Ôºù y) (z : A) ‚Üí x Ôºù z ‚Üí y Ôºù z

  inv-concat' : (x : A) {y z : A} ‚Üí y Ôºù z ‚Üí x Ôºù z ‚Üí x Ôºù y

Properties

Associativity of concatenation

For any three identifications p : x Ôºù y, q : y Ôºù z, and r : z Ôºù w, we have an identification

  assoc p q r : ((p ‚àô q) ‚àô r) Ôºù (p ‚àô (q ‚àô r)).

The identification assoc p q r is also called the {{#concept "associator" Disambiguation="identification" Agda=assoc}}.

Note that the associator assoc p q r is an identification in the type x Ôºù w, i.e., it is an identification of identifications. Here we make crucial use of the fact that the identity types are defined for all types. In other words, since identity types are themselves types, we can consider identity types of identity types, and so on.

Associators

module _
  {l : Level} {A : UU l}
  where

  assoc :
    {x y z w : A} (p : x Ôºù y) (q : y Ôºù z) (r : z Ôºù w) ‚Üí
    ((p ‚àô q) ‚àô r) Ôºù (p ‚àô (q ‚àô r))
  assoc refl q r = refl

The unit laws for concatenation

For any identification p : x Ôºù y there is an identification

  left-unit : (refl ‚àô p) Ôºù p.

Similarly, there is an identification

  right-unit : (p ‚àô refl) Ôºù p.

In other words, the reflexivity identification is a unit element for concatenation of identifications.

module _
  {l : Level} {A : UU l}
  where

  double-assoc :
    {x y z w v : A} (p : x Ôºù y) (q : y Ôºù z) (r : z Ôºù w) (s : w Ôºù v) ‚Üí
    (((p ‚àô q) ‚àô r) ‚àô s) Ôºù p ‚àô (q ‚àô (r ‚àô s))
  double-assoc refl q r s = assoc q r s

  triple-assoc :
    {x y z w v u : A}
    (p : x Ôºù y) (q : y Ôºù z) (r : z Ôºù w) (s : w Ôºù v) (t : v Ôºù u) ‚Üí
    ((((p ‚àô q) ‚àô r) ‚àô s) ‚àô t) Ôºù p ‚àô (q ‚àô (r ‚àô (s ‚àô t)))
  triple-assoc refl q r s t = double-assoc q r s t

Unit laws

  left-unit : {x y : A} {p : x Ôºù y} ‚Üí refl ‚àô p Ôºù p
  left-unit = refl

  right-unit : {x y : A} {p : x Ôºù y} ‚Üí p ‚àô refl Ôºù p
  right-unit {p = refl} = refl

The inverse laws for concatenation

module _

  where

  left-inv : {x y : A} (p : x Ôºù y) ‚Üí inv p ‚àô p Ôºù refl
  left-inv refl = refl

  right-inv : {x y : A} (p : x Ôºù y) ‚Üí p ‚àô (inv p) Ôºù refl

    inv (p ‚àô q) Ôºù inv q ‚àô inv p

Concatenating with an inverse is inverse to concatenating

We show that the operation q ‚Ü¶ inv p ‚àô q is inverse to the operation q ‚Ü¶ p ‚àô q by constructing identifications

  inv p ‚àô (p ‚àô q) Ôºù q
  p ‚àô (inv p ‚àô q) Ôºù q.

Similarly, we show that the operation p ‚Ü¶ p ‚àô inv q is inverse to the operation p ‚Ü¶ p ‚àô q by constructing identifications

  (p ‚àô q) ‚àô inv q Ôºù p
  (p ‚àô inv q) ‚àô q Ôºù p.

In foundation.identity-types we will use these families of identifications to conclude that concat p z and concat' x q are equivalences with inverses concat (inv p) z and concat' x (inv q), respectively.

module _
  {l : Level} {A : UU l}
  where

  is-retraction-inv-concat :
    {x y z : A} (p : x Ôºù y) (q : y Ôºù z) ‚Üí (inv p ‚àô (p ‚àô q)) Ôºù q
  is-retraction-inv-concat refl q = refl

  is-section-inv-concat :
    {x y z : A} (p : x Ôºù y) (r : x Ôºù z) ‚Üí (p ‚àô (inv p ‚àô r)) Ôºù r
  is-section-inv-concat refl r = refl

  is-retraction-inv-concat' :
    {x y z : A} (q : y Ôºù z) (p : x Ôºù y) ‚Üí (p ‚àô q) ‚àô inv q Ôºù p
  is-retraction-inv-concat' refl refl = refl

  is-section-inv-concat' :
    {x y z : A} (q : y Ôºù z) (r : x Ôºù z) ‚Üí (r ‚àô inv q) ‚àô q Ôºù r
  is-section-inv-concat' refl refl = refl

Transposing inverses

Consider a triangle of identifications

      p
  x ----> y
   \     /
  r \   / q
     ‚à® ‚à®
      z

in a type A. Then we have maps

  p ‚àô q Ôºù r ‚Üí q Ôºù inv p ‚àô r
  p ‚àô q Ôºù r ‚Üí p Ôºù r ‚àô inv q.

In foundation.identity-types we will show that these maps are equivalences.

module _
  {l : Level} {A : UU l}
  where

  left-transpose-eq-concat :
    {x y : A} (p : x Ôºù y) {z : A} (q : y Ôºù z) (r : x Ôºù z) ‚Üí
    p ‚àô q Ôºù r ‚Üí q Ôºù inv p ‚àô r
  left-transpose-eq-concat refl q r s = s

  inv-left-transpose-eq-concat :
    {x y : A} (p : x Ôºù y) {z : A} (q : y Ôºù z) (r : x Ôºù z) ‚Üí
    q Ôºù inv p ‚àô r ‚Üí p ‚àô q Ôºù r
  inv-left-transpose-eq-concat refl q r s = s

  right-transpose-eq-concat :
    {x y : A} (p : x Ôºù y) {z : A} (q : y Ôºù z) (r : x Ôºù z) ‚Üí
    p ‚àô q Ôºù r ‚Üí p Ôºù r ‚àô inv q
  right-transpose-eq-concat p refl r s = (inv right-unit ‚àô s) ‚àô inv right-unit

  inv-right-transpose-eq-concat :
    {x y : A} (p : x Ôºù y) {z : A} (q : y Ôºù z) (r : x Ôºù z) ‚Üí
    p Ôºù r ‚àô inv q ‚Üí p ‚àô q Ôºù r
  inv-right-transpose-eq-concat p refl r s = right-unit ‚àô (s ‚àô right-unit)

  double-transpose-eq-concat :
    {x y u v : A} (r : x Ôºù u) (p : x Ôºù y) (s : u Ôºù v) (q : y Ôºù v) ‚Üí
    p ‚àô q Ôºù r ‚àô s ‚Üí inv r ‚àô p Ôºù s ‚àô inv q
  double-transpose-eq-concat refl p s refl Œ± =
    (inv right-unit ‚àô Œ±) ‚àô inv right-unit

  double-transpose-eq-concat' :
    {x y u v : A} (r : x Ôºù u) (p : x Ôºù y) (s : u Ôºù v) (q : y Ôºù v) ‚Üí
    p ‚àô q Ôºù r ‚àô s ‚Üí q ‚àô inv s Ôºù inv p ‚àô r
  double-transpose-eq-concat' r refl refl q Œ± = right-unit ‚àô (Œ± ‚àô right-unit)

Splicing and unsplicing concatenations of identifications

Consider two identifications p : a Ôºù b and q : b Ôºù c, and consider two further identifications r : b Ôºù x and s : x Ôºù b equipped with an identification inv r Ôºù s, as indicated in the diagram

           x
          ‚àß |
        r | | s
          | ‚à®
  a -----> b -----> c.

Then we have identifications

    splice-concat : p ‚àô q Ôºù (p ‚àô r) ‚àô (s ‚àô q)
  unsplice-concat : (p ‚àô r) ‚àô (s ‚àô q) Ôºù p ‚àô q.

module _
  {l : Level} {A : UU l}
  where

  splice-concat :
    {a b c x : A}
    (p : a Ôºù b) {r : b Ôºù x} {s : x Ôºù b} (Œ± : inv r Ôºù s) (q : b Ôºù c) ‚Üí
    p ‚àô q Ôºù (p ‚àô r) ‚àô (s ‚àô q)
  splice-concat refl {r} refl q = inv (is-section-inv-concat r q)

  splice-concat' :
    {a b c x : A}
    (p : a Ôºù b) {r : b Ôºù x} {s : x Ôºù b} (Œ± : r Ôºù inv s) (q : b Ôºù c) ‚Üí
    p ‚àô q Ôºù (p ‚àô r) ‚àô (s ‚àô q)
  splice-concat' refl {.(inv s)} {s} refl q = inv (is-retraction-inv-concat s q)

  unsplice-concat :
    {a b c x : A}
    (p : a Ôºù b) {r : b Ôºù x} {s : x Ôºù b} (Œ± : inv r Ôºù s) (q : b Ôºù c) ‚Üí
    (p ‚àô r) ‚àô (s ‚àô q) Ôºù p ‚àô q
  unsplice-concat p Œ± q = inv (splice-concat p Œ± q)

  unsplice-concat' :
    {a b c x : A}
    (p : a Ôºù b) {r : b Ôºù x} {s : x Ôºù b} (Œ± : r Ôºù inv s) (q : b Ôºù c) ‚Üí
    (p ‚àô r) ‚àô (s ‚àô q) Ôºù p ‚àô q
  unsplice-concat' p Œ± q = inv (splice-concat' p Œ± q)

Concatenation is injective

module _
  {l1 : Level} {A : UU l1}
  where

  is-injective-concat :
    {x y z : A} (p : x Ôºù y) {q r : y Ôºù z} ‚Üí p ‚àô q Ôºù p ‚àô r ‚Üí q Ôºù r
  is-injective-concat refl s = s

  is-injective-concat' :
    {x y z : A} (r : y Ôºù z) {p q : x Ôºù y} ‚Üí p ‚àô r Ôºù q ‚àô r ‚Üí p Ôºù q
  is-injective-concat' refl s = (inv right-unit) ‚àô (s ‚àô right-unit)

Equational reasoning

Identifications can be constructed by equational reasoning in the following way:

equational-reasoning
  x Ôºù y by eq-1
    Ôºù z by eq-2
    Ôºù v by eq-3

The resulting identification of this computaion is eq-1 ‚àô (eq-2 ‚àô eq-3), i.e., the identification is associated fully to the right. For examples of the use of equational reasoning, see addition-integers.

infixl 1 equational-reasoning_
infixl 0 step-equational-reasoning

equational-reasoning_ :
  {l : Level} {X : UU l} (x : X) ‚Üí x Ôºù x
equational-reasoning x = refl

step-equational-reasoning :
  {l : Level} {X : UU l} {x y : X} ‚Üí
  (x Ôºù y) ‚Üí (u : X) ‚Üí (y Ôºù u) ‚Üí (x Ôºù u)
step-equational-reasoning p z q = p ‚àô q

syntax step-equational-reasoning p z q = p Ôºù z by q

Note. Equational reasoning is a convenient way to construct identifications. However, in some situations it may not be the fastest or cleanest mechanism to construct an identification. Some constructions of identifications naturally involve computations that are more deeply nested in the terms. Furthermore, proofs by equational reasoning tend to require a lot of reassociation.

Some tools that allow us to perform faster computations are the transpositions defined above, the transpositions and splicing operations defined in commuting squares of identifications and commuting triangles of identifications, and the higher concatenation operations defined in path algebra. Each of these operations has good computational behavior, so there is infrastructure for reasoning about identifications that are constructed using them.

We also note that there is similar infrastructure for homotopy reasoning.
